# Design Gates

## Check

- Design matches audience, use case, and brand direction.
- Text fits on mobile and desktop.
- Interactive controls have expected states.
- Contrast and keyboard/focus behavior are acceptable for implementation targets.
- Generated bitmap assets are saved in the workspace if referenced.
- Vector/code-native assets are not replaced by raster images unless intended.

## Tools

- Cowart/Figma for visual context.
- `frontend-design` for concrete UI creation, `ui-ux-pro-max` for UX/pattern judgment, and `design-taste-frontend` only when visual taste calibration is needed.
- imagegen for raster assets and visual references.
- Chrome/Playwright for web checks.
- Computer Use for local design apps or export panels without an API path.
