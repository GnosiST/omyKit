# Change Modes

Choose the smallest workflow that can produce a reliable deliverable.

## Lite

Use for small, reversible work.

Shape:
1. One-paragraph brief.
2. Focused context.
3. Direct implementation or artifact creation.
4. Minimum verification.

## Standard

Use by default.

Shape:
1. Brief or OpenSpec-style change.
2. Impact/context check.
3. Short plan with verification per step.
4. Implementation.
5. Focused checks.
6. Delivery gate.

## Strict

Use for durable or high-risk work.

Shape:
1. Constitution/project principles if absent.
2. Explicit spec/design/tasks.
3. CodeGraph or equivalent impact analysis for code.
4. Superpowers planning/debugging/TDD as relevant.
5. Runtime readiness.
6. Full delivery gates and review.
7. Knowledge cleanup after acceptance.

## Superpowers Routing

- Fuzzy idea -> brainstorming.
- Multi-step plan -> writing-plans.
- Existing checklist -> executing-plans.
- Bug/regression -> systematic-debugging.
- Risky logic -> test-driven-development.
- Done claim -> verification-before-completion.
- Review requested -> requesting-code-review.
