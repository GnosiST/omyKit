# 语言策略

语言：[English](language-policy.md) | [简体中文](language-policy.zh-CN.md)

omyKit 应让用户可见的 agent 输出符合用户使用的语言，方便阅读和理解。

## 规则

用户可见语言跟随最新用户提示词：

- 中文提示 -> 中文可见输出。
- 英文提示 -> 英文可见输出。
- 混合提示 -> 使用主导语言，除非用户明确要求另一种语言。
- 代码、命令、文件名、API 名、专有名词和引用原文保持原样。

## 哪些内容需要跟随

以下内容使用选定语言：

- 可见推理摘要
- 计划和进度更新
- 澄清问题
- 假设和取舍
- 验证总结
- 最终 handoff
- 当交付物对语言敏感时，生成的项目文档也应跟随

## 边界

不要暴露内部私有思维链。用户要求“思考过程”或“推理过程”时，输出简洁的推理摘要、决策记录、假设、证据和下一步，并使用用户语言。

## 交接

语言选择不明确时，简短说明假设并继续。如果用户纠正语言，后续用户可见输出立即切换。
